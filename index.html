07 API Spec
API คืออะไร และมี Endpoint อะไรบ้าง?
API (Application Programming Interface) คือ "ประตู" ที่ Frontend ใช้คุยกับ Backend ทุกการกระทำ (ดูสินค้า / สั่งซื้อ / login) ล้วนผ่าน API ทั้งสิ้น
👤 Users — /api/v1/users
MethodPathAuth / Role
POST/registerสาธารณะ
POST/loginสาธารณะ
POST/logout✓ ผู้ใช้ที่ login
GET/profile✓ ตัวเอง
PUT/profile✓ แก้ข้อมูลตัวเอง
PATCH/profile/password✓ เปลี่ยนรหัสผ่าน
GET/Staff | Admin
PATCH/:id/roleAdmin เท่านั้น
DELETE/:idลบ user (Staff | Admin)
📦 Products — /api/v1/products
MethodPathทำอะไร
GET/ดูสินค้าทั้งหมด
GET/number/:productNumberดูตาม productNumber
GET/:idดูตาม id
POST/เพิ่มสินค้าใหม่
PUT/:idแก้ไขสินค้า
DELETE/:idลบสินค้า
🛒 Carts — /api/v1/carts [ต้อง Login]
MethodPathทำอะไร

08 Data Schemas / Models
วิธีเก็บข้อมูลใน MongoDB
Schema คือ "แบบฟอร์ม" ที่กำหนดว่าข้อมูลแต่ละชนิดมีหน้าตาอย่างไร มี field อะไรบ้าง และมีกฎอะไรบ้าง ก่อนบันทึกลง MongoDB
USER Schema
const userSchema = {
  userNumber: { unique, immutable }, // ← Auto-increment
  email: { unique, required },
  password: { select: false }, // ← ซ่อนจาก query
  role: "customer | staff | admin",
  address / shippingAddress / serviceAddress
}
💡 ก่อน save ระบบจะ auto-increment userNumber และ hash password ด้วย bcrypt โดยอัตโนมัติ
PRODUCT Schema
const productSchema = {
  productNumber: { unique, immutable },
  sku: { unique, lowercase },
  specs: [{ label, value }], // สเปคสินค้า
  image: { url, cloudinaryId },
  gallery: [{ url, cloudinaryId }],
  price, salePrice, stock, stockMin
}
💡 cloudinaryId ใช้สำหรับลบรูปจาก Cloudinary ได้ภายหลัง
SERVICE Schema — Auto-generate Number
const serviceSchema = new mongoose.Schema({
  serviceNumber: { unique, immutable }, // "sv20250001"
  serviceType: "install|clean|maintenance",
  status: "request_received|scheduled"
          "|in_progress|completed|rescheduled|cancelled",
  appointmentAt, team, title, description,
  images: [{ url, cloudinaryId }],
  internalNote,
  customer: { userNumber, firstName, lastName,
    phone, email, serviceAddress: { addressLine,
    subdistrict, district, province, postcode } }
}, { timestamps: true });
// pre("save"): "sv" + ปี + padStart(4) → "sv20250001"
💡 customer เก็บเป็น snapshot · images[] เก็บ cloudinaryId สำหรับลบรูปภายหลัง
ORDER Schema
const orderSchema = {
  orderNumber: { unique, immutable },
  status: "open|paid|preparing|shipping|delivered|cancelled",
  items: [{ productNumber, priceAtPurchase, qty }],
  customer: { ...snapshot ข้อมูลลูกค้า }
}
// pre("save"): "ord" + ปี + padStart(4) → "ord20250001"
💡 priceAtPurchase บันทึกราคา ณ วันสั่งซื้อ แม้ราคาสินค้าเปลี่ยนภายหลัง ยอดออเดอร์ก็ไม่เปลี่ยน
COUNTER Schema
MongoDB ไม่มี Auto-increment ในตัว — เราแก้ด้วยการสร้าง Collection ชื่อ Counter ที่เก็บตัวเลขล่าสุดของแต่ละ sequence
import mongoose from "mongoose";

const counterSchema = new mongoose.Schema({
  _id: String,
  seq: { type: Number, default: 0 }
}, { versionKey: false });

export const Counter = mongoose.model("Counter", counterSchema);
💡 เรียกใช้ใน pre("save") hook → ได้เลข userNumber/productNumber/orderNumber ที่ไม่ซ้ำกันเสมอ
CART Schema
const cartSchema = new mongoose.Schema({
  userNumber: { type: Number, unique, immutable, index },
  items: [{
    productNumber: { type: Number, required, index },
    quantity: { type: Number, required, min: 1 }
  }]
}, { timestamps: true });
💡 userNumber เป็น unique → 1 user มีได้ 1 cart เท่านั้น · timestamps บันทึกเวลาอัปเดตอัตโนมัติ
09 How the Team Uses AI
ทีมใช้ AI ในการเรียนและพัฒนา Software ยังไง?
AI ไม่ใช่แค่เครื่องมือสร้างโค้ด แต่คือ "เพื่อน Pair Programmer" ที่ช่วยอธิบาย ช่วยคิด และช่วยตรวจสอบงาน
🗺
1. วางแผนและออกแบบระบบ
  • ให้ AI แนะนำ Tech Stack ที่เหมาะกับ requirement
  • ออกแบบ Database Schema ก่อนเขียนโค้ด
  • สร้าง API Contract ล่วงหน้า
📖
2. เรียนรู้ Concept ผ่านโค้ดจริง
  • ให้ AI อธิบายเกี่ยวกับ Cookie vs localStorage
  • เรียน Mongoose Hooks และ Schema validation
  • เข้าใจ JWT Auth Flow ทีละขั้นตอน
🔍
3. Debug และ Code Review
  • Copy error message ให้ AI วิเคราะห์
  • ให้ AI หา edge case ที่มองข้าม
  • ตรวจสอบ Security Issue ก่อน deploy
⚛
4. แปลง HTML → React
  • ให้ AI แนะนำว่าแยก Component ตรงไหน
  • cart.html → CartPage + CartItem + OrderSummary
  • payment.html → PaymentPage + PaymentOption + Modal
📝
5. เขียน API Docs
  • ให้ AI สรุป API Spec จากโค้ดจริง
  • สร้าง .rest file สำหรับทดสอบ API
  • สรุป summary.md เพื่อทำ Slides
✅
หลักการใช้ AI อย่างถูกต้อง
  • ให้ AI อธิบายโค้ดทุกครั้งก่อน copy
  • ตรวจสอบ output เสมอ — AI ผิดได้
  • ใช้เป็น Pair Programmer — ถาม ตอบ ปรับ
  • เรียนจาก Process ไม่ใช่แค่ผลลัพธ์
สิ่งที่ทีมเรียนรู้ได้จากการใช้ AI ในโปรเจกต์นี้
